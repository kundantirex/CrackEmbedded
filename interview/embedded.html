Part 1 30 Core Questions

01 What is an Embedded System?

A small computer built to do ONE specific job inside a bigger product. It is not a general PC — you cannot install random software on it. It quietly runs the same task forever.

ExamplesWashing machine controller, car engine ECU, smart watch, and your Tirex EV charger — all are embedded systems.
Interview tipContrast it with a PC: 'A PC does many things; an embedded system does one thing reliably.' That one line usually lands well.

02 What is Firmware?

Firmware is the software that lives permanently inside the chip's non-volatile memory (Flash) and directly controls the hardware. It stays even after power off. 'Firm' = between hardware and normal software.

ExampleThe code you flash into an ARM Cortex-M that decides when to turn on a relay is firmware.

03 Difference between Hardware and Software?

Hardware is the physical part you can touch (chip, PCB, resistor). Software is the set of instructions that tells the hardware what to do. Hardware is the body, software is the brain telling it to move.

04 What is a Microcontroller?

A microcontroller (MCU) is a full tiny computer on a single chip. Inside one chip it has CPU + memory (RAM/Flash) + peripherals (GPIO, UART, ADC, timers). You add power and a few components and it runs.

ExampleSTM32, MSPM0, i.MX RT — all are MCUs used in EV chargers.

05 Microcontroller vs Microprocessor?

Feature Microcontroller (MCU) Microprocessor (MPU)
Architecture CPU + RAM + Flash + peripherals on ONE chip Only CPU on chip
External Memory Needs external RAM/Flash? No Needs external RAM, Flash, peripherals
Use Case Low power, low cost, real-time High performance, runs an OS (Linux)
Examples STM32, MSPM0 i.MX93, Raspberry Pi CPU
Interview tipYou can say: 'MCU is self-contained; MPU needs external memory but gives more power, like the i.MX93 running Linux.'

06 What is GPIO?

GPIO = General Purpose Input Output pins. Each pin can either READ a digital signal (button, sensor = input) or DRIVE a digital signal (LED, relay = output). Only two states: HIGH (1) or LOW (0).

// Set pin as output, then drive it HIGH gpio_set_direction(LED_PIN, OUTPUT); gpio_write(LED_PIN, HIGH); // LED ON gpio_write(LED_PIN, LOW); // LED OFF

07 What is an Interrupt?

An interrupt is an urgent signal that tells the CPU: 'Stop what you are doing, handle this NOW, then go back.' The special function that runs is called the ISR (see Q26).

Real-life analogyYou are reading a book (main code). The phone rings (interrupt). You pause, answer it (ISR), then continue reading from the same line.

08 Polling vs Interrupt?

Polling = CPU keeps checking again and again 'is it ready? is it ready?' — wastes CPU time. Interrupt = CPU does other work and is notified only when the event actually happens — efficient.

// POLLING while (1) { if (button_pressed()) // checked continuously do_action(); } // INTERRUPT - runs ONLY when button is pressed void button_ISR(void) { do_action(); }
Interview tipSay: 'Polling is simple but wastes power; interrupt is efficient — important in low-power battery devices.'

09 What is a Timer?

A timer is a peripheral that counts clock pulses. Because the clock speed is fixed, counting pulses = measuring time. Used for delays, timeouts, and generating events at fixed intervals (periodic interrupts).

ExampleBlink an LED every 1 second, or send charger status every 10 seconds — a timer handles the timing accurately without blocking the CPU.

10 What is PWM?

PWM = Pulse Width Modulation. It is a square wave that switches ON and OFF very fast. By changing how long it stays ON (the 'duty cycle'), you control the average power delivered.

Example50% duty = ON half the time = half brightness/speed. 20% duty = dim LED or slow motor.
set_pwm_duty(LED, 50); // 50% -> half brightness set_pwm_duty(MOTOR, 80); // 80% -> fast speed

11 What is UART?

UART = Universal Asynchronous Receiver Transmitter. A simple 2-wire serial protocol (TX to send, RX to receive). 'Asynchronous' means no separate clock wire — both sides just agree on a speed called baud rate (e.g. 115200).

ExampleSending debug logs from your MCU to a PC terminal uses UART.

12 Difference between UART and SPI?

Interview tipMemory hook: UART is for talking to a PC/modem; SPI is for fast chips like flash and displays.

13 What is I2C?

I2C = a 2-wire serial protocol: SDA (data) and SCL (clock). Many devices share the same 2 wires, and each device has a unique address. Great when you have several sensors and want to save pins.

ExampleA temperature sensor, an RTC, and an EEPROM can all sit on the same two I2C wires, each answered by its address.

14 What is ADC?

ADC = Analog to Digital Converter. The real world is analog (voltage, temperature, current). The CPU only understands numbers. ADC converts an analog voltage into a digital number the code can use.

uint16_t raw = adc_read(CHANNEL_0); // e.g. 0..4095 (12-bit) float voltage = (raw / 4095.0f) * 3.3f; // convert to volts
Interview tipYou debugged ADC on the MSPM0 — a strong point. Mention polling vs interrupt mode and sequence mode for multi-channel reads if the interviewer digs deeper.

15 What is the purpose of DAC?

DAC = Digital to Analog Converter — the exact opposite of ADC. It turns a digital number from the CPU into a real analog voltage or current output.

ExampleGenerating an audio waveform or a smooth reference voltage from code uses a DAC.

16 What is DMA?

DMA = Direct Memory Access. It moves data between memory and peripherals WITHOUT bothering the CPU. The CPU sets it up once, then is free to do other work while DMA copies data in the background.

Why it mattersMoving a big ADC buffer or UART stream byte-by-byte with the CPU is slow. DMA does it automatically = faster and lower CPU load.

17 What is a Watchdog Timer?

A safety timer. If firmware hangs or gets stuck in a loop, the watchdog resets the whole system automatically. Your code must 'feed' (refresh) it regularly; if it stops feeding, the watchdog assumes a crash and reboots.

while (1) { do_work(); watchdog_feed(); // must reach here in time, else RESET }
Interview tipGreat safety talking point for EV chargers: 'If the main loop hangs, the watchdog reboots the charger to a safe state.'

19 What is Flash Memory?

Flash is non-volatile memory that can be electrically erased and rewritten. Your firmware is stored in Flash. It keeps data without power, but is erased in blocks (not single bytes).

ExampleWhen you flash a .hex/.bin into your MCU, it goes into Flash. The QSPI flash (W25Q64) you worked with is external flash.

20 What is EEPROM?

EEPROM is also non-volatile, but its special power is that you can read and write ONE byte at a time (Flash erases whole blocks). Perfect for small settings that change often.

ExampleSaving a charger's serial number, calibration value, or last-used settings so they survive a reboot.

21 Difference between Stack and Heap?

Both are areas of RAM. Stack is automatic and fast (function calls, local variables) and works in LIFO order — Last In, First Out. Heap is manual memory you request at runtime with malloc and must free yourself.

void func(void) { int x = 5; // STACK: auto-freed on return int *p = malloc(100); // HEAP: you must free it free(p); // else memory leak! }
Interview tipOn MCUs say: 'We avoid heap/malloc in firmware because of fragmentation; we prefer static or stack allocation.' Interviewers love this.

22 What is the 'volatile' keyword in C?

'volatile' tells the compiler: 'This variable can change on its own, outside normal code flow — do NOT optimize it away, read it fresh every time.' Needed for hardware registers, ISR-shared variables, and memory-mapped I/O.

volatile uint8_t flag = 0; // changed inside an ISR void ISR(void) { flag = 1; } while (!flag) { } // WITHOUT volatile, compiler may loop forever
Interview tipClassic must-know question. One-liner: 'volatile stops the compiler from caching a value that hardware or an ISR can change.'

23 What is the 'static' keyword in C?

'static' has two meanings. Inside a function: the variable keeps its value between calls (not reset each time). At file level: it limits the variable/function scope to that file only (private).

void counter(void) { static int count = 0; // keeps value across calls count++; // 1, 2, 3, ... }

24 What is the 'const' keyword in C?

'const' makes a variable read-only after it is set. If code tries to change it, the compiler gives an error. It protects values from accidental modification and documents intent.

const float PI = 3.14159f; PI = 3.2f; // ERROR: cannot modify a const

25 What is a State Machine?

A design pattern where the system is always in exactly ONE state, and it moves to another state based on events. Keeps complex behaviour clean and predictable — the backbone of most firmware.

// EV charger states example switch (state) { case IDLE: if (plug_in) state = CONNECTED; break; case CONNECTED: if (start) state = CHARGING; break; case CHARGING: if (stop) state = IDLE; break; }
Interview tipTie it to your work: 'An OCPP charging session is naturally a state machine — Available, Preparing, Charging, Finishing.'

26 What is an ISR?

ISR = Interrupt Service Routine. The special function that runs automatically when an interrupt fires. Rule: keep it SHORT and FAST — just set a flag or read a value, and do heavy work in the main loop.

volatile bool data_ready = false; void UART_RX_ISR(void) { buffer[i++] = read_byte(); // short & fast data_ready = true; // main loop does the rest }
Interview tipSay what you should NOT do in an ISR: no delays, no printf, no malloc, no long loops.

27 What is the Bootloader?

A small program that runs FIRST at power-on, before the main firmware. It sets up basic hardware and then loads/starts the main application. It can also update firmware (a key feature).

ExampleYour MCU OTA update tool relies on a bootloader: it receives the new firmware over UART/USB, writes it to Flash, then jumps to the new app.

28 What are Low Power Modes?

Modes that reduce power by switching off or slowing down unused parts of the chip (CPU, clocks, peripherals). The MCU 'sleeps' and wakes up on an event like an interrupt. Essential for battery devices.

Common modesSleep (CPU off, peripherals on), Stop/Deep-sleep (most clocks off), Standby (lowest power, almost everything off). Deeper sleep = less power but slower wake-up.

29 What is RTOS and its benefits?

RTOS = Real-Time Operating System. It lets firmware run many tasks 'at once' by quickly switching between them (scheduling), with predictable timing. You split the program into tasks with priorities.

BenefitsHandles many jobs cleanly (read sensor, run UI, send data), guarantees high-priority tasks run on time, and gives ready tools like task delays, queues, and semaphores instead of one giant while-loop.
Interview tipName-drop one: 'I use FreeRTOS with tasks, queues, and semaphores.' Contrast with bare-metal (super-loop) if asked.

30 Embedded C Best Practices?

Habits that make firmware safe and readable:

  • Use meaningful names (chargerState, not x).
  • Avoid global variables; if needed, keep them minimal and controlled.
  • Always check return values and error codes.
  • Use fixed-width types (uint8_t, uint32_t) — sizes are guaranteed.
  • Use const and volatile correctly.
  • Keep ISRs short; avoid malloc/heap in firmware.
  • Comment the WHY, not the obvious WHAT.
Interview tipAdd: 'We follow MISRA-C style guidelines and use a watchdog for safety' — shows real-world discipline.

Part 2 Intermediate & Advanced Topics

A. Communication Protocols (Deep Dive)

A1. I2C — the full picture

I2C is a 2-wire, synchronous, half-duplex protocol from Philips/NXP. Two shared wires — SDA (data) and SCL (clock) — connect many devices. Each device has a unique address. Both lines are open-drain, so they need pull-up resistors (2.2k–10k) to VCC.

  • START: master pulls SDA HIGH→LOW while SCL is HIGH (begins a transfer).
  • STOP: SDA goes LOW→HIGH while SCL is HIGH (ends a transfer).
  • ACK (0): receiver pulls SDA LOW = 'got it'. NACK (1): SDA stays HIGH = wrong address, busy, or end of read.
  • Clock stretching: a slow slave holds SCL LOW to pause the master until it is ready.
  • Arbitration (multi-master): if two masters talk at once, the one sending 0 (dominant) wins; the other sees the mismatch and backs off. No data is corrupted.
// WRITE: START -> Addr+W -> ACK -> Data -> ACK -> STOP // READ : START -> Addr+R -> ACK -> Data -> ACK ... -> NACK -> STOP i2c_read_reg(0x68, 0x3B); // read reg 0x3B from device at addr 0x68
Interview tipWhy open-drain? It allows many devices on one bus without shorting, and enables the wired-AND logic that makes arbitration and clock stretching work. Classic follow-up: 'ACK = 0, NACK = 1.'

A2. SPI — the full picture

SPI is a synchronous, full-duplex protocol for fast, short-distance links. One master, one or more slaves. Data goes both ways at the same time on separate wires.

The 4 signals: SCK (clock), MOSI (Master Out Slave In), MISO (Master In Slave Out), SS/CS (Slave Select - active LOW).

Mode (CPOL,CPHA)Clock idleData sampled on
0 (0,0)LowRising edge
1 (0,1)LowFalling edge
2 (1,0)HighFalling edge
3 (1,1)HighRising edge
cs_low(SLAVE1); // select slave uint8_t rx = spi_transfer(0x9F); // send & receive in one shot cs_high(SLAVE1); // deselect
Interview tipPro tip: always match CPOL/CPHA between master and slave, or you read garbage. SPI vs I2C: SPI is faster and full-duplex but uses more pins and has no addressing (uses CS lines instead).

A3. CAN Bus — the automotive/EV backbone

CAN (Controller Area Network) is a robust, multi-master, differential 2-wire bus (CAN_H, CAN_L) used to connect ECUs without a central computer. Priority is decided by the message ID: lower ID = higher priority. A 0 bit is 'dominant' and overwrites a 1 bit ('recessive') — that is how arbitration works.

CAN 2.0A (Standard)CAN 2.0B (Extended)
11-bit identifier (2,048 IDs)29-bit identifier (536 million IDs)
IDE bit = 0IDE bit = 1
Passenger cars, sensorsBMS, EVs, SAE J1939 trucks
// BMS -> VCU, ID 0x18FF50E5, Data: 0F 91 00 7D ... // Byte0-1 = 0x0F91 = 3985 -> 398.5 V (battery voltage) // Byte2-3 = 0x007D = 125 -> 125 A (battery current)
Interview tipStrong EV talking point: 'Lower CAN ID wins arbitration because dominant 0 overwrites recessive 1; we use extended 29-bit IDs for BMS/charger messages, similar to SAE J1939.'

A4. UDS + ISO-TP over CAN

Think of it as a road and a language: CAN is the road (moves the bytes), UDS is the language spoken on it (what the bytes mean — a diagnostic service). CAN just transports; it doesn't understand the request.

ISO-TP for big messages: A single classic CAN frame carries only 8 bytes. If the answer (like a VIN) is bigger, ISO-TP (ISO 15765-2) automatically splits it into a First Frame (FF), Flow Control (FC), and Consecutive Frames (CF), and the tester reassembles them.

Interview tipOne-liner: 'CAN transports, ISO-TP segments, UDS defines the service. Together they are the foundation of vehicle diagnostics.'

B. Core Peripherals & Embedded C

B1. GPIO & Pull-up / Pull-down resistors

Inside the MCU each GPIO has a Direction register (0 = input, 1 = output), a Data (output) register, and an Input register. As output you write the data register to drive the pin HIGH/LOW; as input you read the input register to sense the level.

The floating-pin problem: An input pin connected only to an open switch is 'floating' — it picks up noise and reads random 0s and 1s. A pull resistor fixes the pin to a known level when the switch is open.

ResistorSwitch OPEN (idle)Switch PRESSED
Pull-up≈ VCC → HIGH (1)≈ 0V → LOW (0)
Pull-down≈ 0V → LOW (0)≈ VCC → HIGH (1)

B2. ADC in depth

An ADC converts a real analog voltage into a digital number. It works in 4 steps: Sample (grab the value), Hold (keep it steady), Quantize (snap to nearest level), Encode (output binary).

// The two formulas you must know: LSB (step size) = Vref / 2^n // smallest voltage it can see Digital code = Vin / LSB // the number you read // Worked example (Vref = 3.3 V, 12-bit, Vin = 1.65 V) LSB = 3.3 / 4096 = 0.000805 V // (0.805 mV) code = 1.65 / 0.000805 = 2048 // (half scale, as expected)

B3. Bitwise operators in C

Firmware constantly sets/clears single bits in hardware registers. These operators work on the binary bits of a value.

// The 4 things you do to a register reg |= (1 << n); // SET bit n to 1 reg &= ~(1 << n); // CLEAR bit n to 0 reg ^= (1 << n); // TOGGLE bit n if (reg & (1 << n)) // CHECK if bit n is 1 val = reg & MASK; // MASK isolate specific bits
Interview tipThis is the single most-used skill in firmware. Memorise set/clear/toggle/check by heart — interviewers often ask you to write them on the spot.

C. CPU Efficiency & Timing

C1. Polling vs Interrupt vs DMA

MethodHow it worksCPU Load
PollingCPU keeps checkingHigh (wastes cycles)
InterruptDevice notifies CPUModerate
DMAHardware moves dataVery low

One-liner: polling checks, interrupts notify, DMA transfers.

C2. Quick-fire concepts

  • Run code from Flash vs SRAM: Flash is non-volatile (keeps code); SRAM is volatile (used for runtime data).
  • XIP (Execute-In-Place): CPU runs code directly from Flash without copying to RAM. Saves RAM and boot time.
  • SysTick: A built-in timer in ARM Cortex-M that fires periodic interrupts. Used for RTOS scheduling.
  • Clock gating: Turns off the clock to unused peripherals to save power.

D. Bootloader & Firmware Update

D1. Bootloader & the update process

A bootloader is a small protected program that runs FIRST at power-on. It decides: run the existing app, or enter update mode to receive new firmware.

POWER ON -> BOOTLOADER -> Update requested? NO -> jump to Application (firmware) YES -> receive new FW -> verify (CRC) -> write to Flash -> jump

Windowed vs Non-windowed Watchdog: Non-windowed can be refreshed any time before timeout. Windowed must be refreshed only inside a specific time window (more secure, detects early AND late refresh).

Interview tipDirect match to your MCU OTA tool: mention verify-before-jump, a backup image for rollback, and keeping the bootloader in a protected region.

E. RTOS & Task Scheduling

E1. Task scheduling

Scheduling decides which task runs, when, and for how long. Two families:

  • Cooperative: each task voluntarily gives up the CPU. Simple, low overhead, but can block everything.
  • Preemptive: a higher-priority task can interrupt a lower one. Fast response, ideal for real-time.

Task states: READY → RUNNING → BLOCKED/WAITING → back to READY. The scheduler always runs the highest-priority READY task.

E2. RTOS synchronization mechanisms

  • Mutex: a lock with ownership. Best for protecting shared data.
  • Semaphore: a signaling counter. Binary = signal an event; Counting = manage N resources.
  • Queue: a FIFO mailbox — one task sends messages, another receives them.
  • Event flags: a set of bits where a task can wait for one or several events to occur.
xSemaphoreTake(mutex, portMAX_DELAY); // lock shared_value++; // safe critical section xSemaphoreGive(mutex); // unlock
Interview tipThe classic question: mutex vs semaphore? Mutex has ownership and is for protecting a resource; a semaphore is for signaling/counting and has no owner.

F. Device Drivers (Bare-metal & Linux)

F1. What is a device driver?

A driver is a software layer that sits between the application and a hardware peripheral. The app calls simple functions; the driver does the register-level work underneath.

F2. Linux driver model & loading lifecycle

In Linux, three things bind together: a Device, a Driver, and a Bus. The kernel matches a driver to a device, then calls the driver's probe() to set it up. remove() cleans up when it unbinds.

// Loading lifecycle 1. Write driver.c (define probe, remove, file_ops) 2. make -> driver.ko 3. sudo insmod driver.ko 4. Kernel matches device (Device Tree / ACPI / USB IDs) 5. probe() runs: request GPIOs/IRQs, map I/O 6. /dev/mydevice node created 7. User space uses open/read/write on /dev/mydevice 8. sudo rmmod driver -> remove() frees memory, IRQs
Interview tipDirect match to your i.MX93 embedded-Linux work: mention Device Tree matching, probe()/remove(), and 'proper cleanup in remove() prevents resource leaks.'
