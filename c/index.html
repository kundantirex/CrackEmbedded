Basics of C

Structure of a C program

A C program typically contains: Preprocessor Directives (#include), Global Variables, int main() function, and other User-Defined Functions.

Tokens, Keywords, Identifiers

Token: The smallest individual unit in a C program (e.g., keywords, identifiers, constants).
Keyword: Reserved words with fixed meaning (e.g., int, return, if). C has 32 standard keywords.
Identifier: User-defined names for variables, functions, arrays. Must start with a letter or underscore.

Data Types & Modifiers

Basic types: char (1 byte), int (4 bytes), float (4 bytes), double (8 bytes), void (empty).
Modifiers: signed (can be negative), unsigned (positive only, extends range), short (2 bytes), long (8 bytes).

Input/Output & Format Specifiers

printf() displays output. scanf() reads input from the user.
Format Specifiers: %d (int), %f (float), %c (char), %s (string), %x (hexadecimal), %p (pointer).

Operators

Operators perform operations on variables and values.

  • Arithmetic: +, -, *, /, % (modulo/remainder).
  • Relational: ==, !=, >, <, >=, <= (returns 1 for true, 0 for false).
  • Logical: && (Logical AND), || (Logical OR), ! (Logical NOT).
  • Increment/Decrement: ++ (adds 1), -- (subtracts 1). Can be pre (++x) or post (x++).
  • Bitwise: & (AND), | (OR), ^ (XOR), ~ (NOT), << (Left Shift), >> (Right Shift).
  • Conditional (Ternary): condition ? if_true : if_false.
  • sizeof: Returns the size of a variable or data type in bytes.

Control Statements

Control statements change the normal sequential flow of execution.

  • if / if-else: Executes a block of code if the condition is true; else executes another block.
  • switch: Selects one of many blocks of code to be executed based on an integer/char value. Faster than many if-else chains.
  • for loop: Best when you know exactly how many times you want to loop. for(init; condition; increment)
  • while loop: Best when you don't know the exact number of iterations. Checks condition before running.
  • do-while loop: Guaranteed to run at least once because the condition is checked at the end.
  • break: Exits the current loop or switch immediately.
  • continue: Skips the rest of the current loop iteration and moves to the next one.

Functions

Declaration, Definition, Prototype

Prototype (Declaration): Tells the compiler the function name, return type, and parameters before it is used. e.g. int add(int, int);
Definition: The actual body and logic of the function.

Call by Value

Passing a copy of a variable to a function. Changing the variable inside the function does not change the original variable outside.

Recursion

When a function calls itself. It must have a "base case" to stop calling itself, otherwise it causes a Stack Overflow.

Inline Functions

Keyword inline requests the compiler to expand the function code exactly where it is called, saving function-call overhead. Good for very small, frequently used functions.

Arrays & Strings

Arrays

A collection of elements of the same data type stored in contiguous memory locations. When you pass an array to a function, it decays into a pointer to its first element (Call by Reference).

Strings

A string in C is just a 1D character array terminated by a Null character ('\0').

string.h functions: strlen() (length), strcpy() (copy), strcat() (append), strcmp() (compare).

char[] vs char*:
char str[] = "hello"; is stored in modifiable read-write memory (stack).
char *str = "hello"; points to a read-only string literal in the data segment. Modifying it causes a segmentation fault.

Pointers

A pointer is a variable that stores the memory address of another variable.

  • Pointer arithmetic: Adding 1 to an int * increases the memory address by 4 bytes (the size of an int).
  • Void pointer (void *): A generic pointer that can point to any data type. Must be typecast before dereferencing.
  • NULL pointer: A pointer pointing to nothing (address 0).
  • Wild pointer: An uninitialized pointer that points to a random memory location.
  • Dangling pointer: A pointer that points to a memory location that has been deleted or freed.
  • Function pointer: Points to executable code (a function). Used for callbacks.

Structures, Unions, Enum

Structures

Groups different data types under one name. Structure Padding: The compiler adds empty bytes between struct members to align them in memory for faster CPU access.

Unions

Similar to structures, but all members share the same memory location. The size of the union is the size of its largest member. You can only use one member at a time.

Bit Fields

Allows you to specify the exact number of bits a struct member should use, saving memory. e.g., int flag : 1; uses exactly 1 bit.

Enum (Enumeration)

A user-defined data type that consists of named integer constants. Improves code readability. Default starts at 0.

Preprocessor

Code that runs before compilation.

  • #include: Copies the contents of a header file into the source file.
  • #define: Defines a macro (a text replacement rule).
  • Macros vs Inline: Macros are dumb text replacement done by the preprocessor (can cause side effects like MACRO(x++)). Inline functions are type-checked by the compiler.
  • Conditional Compilation: #ifdef, #ifndef, #endif. Used to include/exclude parts of code based on conditions (e.g., debugging).
  • Header Guards: #ifndef HEADER_H #define HEADER_H ... #endif prevents a header file from being included multiple times in the same file.

File Handling

Allows C programs to read/write persistent data to the disk.

  • fopen(): Opens a file in read (r), write (w), or append (a) mode.
  • fclose(): Closes the file to save data and release resources.
  • Text vs Binary: Text files store human-readable ASCII. Binary files (mode "rb", "wb") store raw bytes (like structs or images), which is faster and smaller.
  • fprintf() / fscanf(): Write and read formatted text.
  • fwrite() / fread(): Write and read raw binary data blocks.

Advanced C Concepts

  • const: Makes a variable read-only. const int *p (pointer to const value) vs int * const p (const pointer to value).
  • volatile: Tells the compiler the variable can change unpredictably (e.g., hardware register, ISR). Prevents compiler optimization.
  • restrict: A promise to the compiler that the pointer is the ONLY way to access that memory. Allows heavy optimization.
  • Endianness: Little-Endian stores the least significant byte first (Intel/ARM). Big-Endian stores the most significant byte first (Network protocols).
  • Undefined Behavior (UB): Code that the C standard does not define (e.g., dividing by zero, out-of-bounds array). The compiler can do literally anything (crash, format your drive).

Frequently Asked Interview Questions

  • malloc vs calloc: malloc allocates raw memory with garbage values. calloc allocates memory AND initializes it to zero.
  • static vs extern: static hides a variable inside its current file. extern tells the compiler the variable exists in another file.
  • struct vs union: Struct allocates separate memory for every member. Union allocates one shared block of memory equal to the largest member.
  • const vs volatile: const means "I won't change this". volatile means "Hardware might change this unexpectedly, don't cache it." They can be used together (e.g., read-only hardware status register).
  • Why main returns int? It returns an exit status to the Operating System. 0 means success, anything else means an error occurred.
  • What is a Segmentation Fault? An error caused by a program trying to access memory that it does not have permission to read/write (e.g., dereferencing a NULL or dangling pointer).
